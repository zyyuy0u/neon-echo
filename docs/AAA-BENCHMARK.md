# AAA 對標審計與改造計畫（M8）

> 2026-07-18。對標對象：薩爾達曠野之息/王國之淚（相機、世界引導、聖所演出）、
> 原神（HUD/羅盤、選單質感、開放世界 QoL）、一般 3A 標準（juice、可及性）。
> 原則：對標的是**設計語言**，不是資產量；每項落地都必須過既有品質閘（測試/預算/驗收）。

## 實施批次（每批走完整收貨流程：實跑 e2e → 截圖親審 → fresh 驗收 → commit）

### M8a — 手感與開場（P0 全部）
1. 相機輔助：跟隨延遲（位置 lerp 分軸）、移動 2s 後自動回背後（可設定關閉）、
   衝刺 FOV +6°、落地相機微沉（幅度隨落速，減少動態=0）。
2. 落地演出：塵埃粒子環 + 模型 squash-stretch（0.12s）+ 落地音（落速分兩檔）。
3. 腳步聲：Run clip 節奏同步的程序化合成腳步（兩音色交替，音量隨速度）。
4. 開場：黑幕 3s 淡入 + 「回聲行者，醒來」字幕 + 鏡頭從仰角緩降到追尾位；
   情境教學提示：首次可跳缺口前 3m 顯示「Space 跳躍」、取得衝刺後首次顯示「Shift 衝刺」
   （每提示只出現一次，入存檔）。
5. 首個目標引導：開場後地面導引線亮起脈衝流向南區（已有導引線，加流動動畫）。

### M8b — 探索 QoL（P1 全部）
1. 羅盤條 HUD（原神式）：頂部橫條，N/E/S/W + 四地標圖示 + 聖所圖示（已發現才顯示）
   + 距離公尺數；與減少動態相容（關動畫保功能）。
2. 快速傳送：暫停選單「傳送」頁——靜滯廣場 + 已完成聖所為錨點；傳送有 0.8s
   光柱淡出/淡入演出；入存檔。
3. 聖所完成 fanfare：輸入凍結 1.2s + 祭壇光柱 + 上行琶音 sting + 能力圖示放大入 HUD
   （減少動態=僅 sting+圖示）。
4. UI 音效：選單移動/確認/返回三音 + 碎片計數 tick。

### M8c — 氛圍與演出（P2 全部）
1. 天色循環：12 分鐘一輪的色相偏移（合成波日落⇄深夜藍紫，不出現白晝），
   霧色/太陽亮度聯動；存檔記時間相位。
2. 碑文演出：逐字顯示（每字 18ms，可跳過）+ 音節 blip；開啟時鏡頭輕推近 8%。
3. 選單動效：面板滑入 240ms、按鈕 hover/focus 輝光脈衝、主選單背景加緩慢粒子。
4. 結局演出強化：喚醒——全城燈光波次點亮由近而遠 4s；安眠——燈光由遠而近熄滅。

## 驗收原則
- 每批 lint/typecheck/test/build 全綠；e2e 全綠且預算斷言不變（drawCalls<300）。
- 新演出全部接「減少動態」開關（a11y 不得倒退）。
- 手感參數不動（僅相機/演出層）；每批新增單元測試覆蓋純邏輯（羅盤方位計算、
  傳送資格、教學提示一次性、天色相位）。
