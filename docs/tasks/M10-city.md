# M10 派工單：城市化改造——「非常像城市，不可以只有形狀」

## GOAL（目標與背景）
User 明確指令：現在的世界是霓虹幾何抽象體，要改造成**一眼就認得出的城市**。
「不可以只有形狀」= 表面必須有城市語彙：**窗格、招牌、車道線、路燈、天際線**。
基線：最新 main（M9 全簽收）。美學仍是合成波霓虹（palette 不變），但內容從抽象變具象。

### 硬性設計要求（每一條都是驗收）
1. **建築必須有窗**：程序化 canvas 貼圖生成窗格矩陣（隨機 30–60% 窗點亮、
   暖橙/青兩色調、少數閃爍），以 emissive map 貼上建築體。禁止無窗的純色大樓。
   建築屋頂要有細節：天線/水塔/空調箱/霓虹屋頂燈條（實例化小幾何,至少兩種輪廓變化）。
   建築本體多樣性：至少 6 種 footprint×高度組合、部分退縮式樓層（setback）。
2. **街道系統**：主要可行走地面改為街道語彙——柏油暗面 + **車道中線/邊線**
   （canvas 貼圖或 shader）、斑馬線於路口、人行道與路緣石（高度差 0.15，不影響
   collider——視覺浮貼）。既有網格美學保留於「廣場/平台」類區域,街道用道路貼圖。
3. **招牌與霓虹看板**：程序化 canvas 生成發光招牌（假字符/幾何圖樣+邊框,
   ≥8 種變化）,掛在建築立面與街道上方；1–2 個大型動畫看板（uv 捲動）。
   全圖招牌總數 ≥40（實例化）。
4. **街道道具**：路燈（發光頭+燈桿,街道沿線實例化,全圖 ≥60）、
   紅綠燈或號誌 ≥10、長椅/垃圾桶/消防栓任兩種 ≥20。
5. **天際線**：地圖邊界外圍一圈背景建築剪影（帶窗格貼圖,兩層深度,實例化）,
   讓每個方向的地平線都是城市輪廓,不是空蕩霧面。
6. **城市生命感**：3–5 條飛行器光軌（發光點沿預設路徑巡航,拖尾）;
   reduced-motion=靜止光點。
7. **四大地標城市化**：保持剪影可辨識,但表面加窗格/結構語彙（天梯=站體+軌道、
   巨塔=窗格塔身、倒懸環=結構桁架、深谷=谷壁燈窗）。

## 工程約束
- **碰撞體與玩法佈局一律不變**（WorldBuilder 的 collider 生成路徑不准動）;
  城市化是視覺層：替換材質/貼圖、增加**非碰撞**裝飾網格。
- 貼圖全程序化（canvas 2D 生成 → CanvasTexture）,禁止外部圖片資產;
  貼圖生成函式（窗格矩陣/招牌/道路）為**純函式可測**（給定 seed→確定輸出,
  以像素抽樣斷言測試）。
- 效能紅線不變：drawCalls < 300、triangles < 500000（實例化+合併是必須手段,
  裝飾物全部進 InstancedMesh 或 merged geometry）;貼圖總記憶體 ≤64MB。
- 世界資料檔只准**新增**裝飾欄位;tuning 手感值不准動;既有測試斷言不准改弱。
- 不准新增依賴;不准 commit;不准跑 `npm run e2e`。

## ACCEPTANCE（驗收條件）
1. `npm run lint`、`npm run typecheck`、`npm test`、`npm run build` 全部 exit 0。
2. 貼圖生成單元測試：窗格貼圖（給定 seed 之下點亮比例在 30–60% 區間、兩色調存在——
   像素抽樣統計斷言）;招牌貼圖 ≥8 變化（不同 seed 輸出不同——雜湊比對）;
   道路貼圖含中線色帶（指定行像素斷言）。
3. 裝飾數量斷言入測試（讀實際場景建構輸出或資料）：招牌 ≥40、路燈 ≥60、
   號誌 ≥10、天際線背景建築 ≥30、飛行器路徑 3–5。
4. `e2e/city.spec.ts`（新）：載入 → drawCalls < 300 且 triangles < 500000（城市化後
   仍守住）→ `getWorldStats()` 擴充回報裝飾統計並斷言 ≥門檻 → 傳送至街道視角/
   建築近景/天際線視角三個點各截圖 → console 零 error。
5. 既有十二條 e2e 全綠（主控執行）。
6. **主控多視角截圖美術審查為最終閘**（廣場街景/建築近景/塔頂俯瞰/天際線,
   審「像不像城市」——過不了就退件重做,標準見上方硬性設計要求）。
